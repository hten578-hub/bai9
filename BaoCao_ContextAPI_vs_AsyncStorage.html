React Native Navigation: Context API vs AsyncStorage

Môn học: Lập trình Di động  |  Họ tên: [Tên sinh viên]  |  MSSV: [MSSV]

1. Context API và AsyncStorage khác nhau như thế nào?

Đây là hai công cụ phục vụ mục đích hoàn toàn khác nhau trong React Native:

Tiêu chí Context API AsyncStorage
Bản chất Cơ chế quản lý state toàn cục của React Hệ thống lưu trữ key-value trên thiết bị
Vị trí lưu RAM (bộ nhớ tạm) Bộ nhớ vĩnh viễn của thiết bị (disk)
Tắt app Mất toàn bộ dữ liệu Dữ liệu vẫn còn
Tốc độ Rất nhanh (đồng bộ) Chậm hơn (bất đồng bộ, I/O)
Cú pháp Đồng bộ, dùng hook useContext Bất đồng bộ, dùng async/await
Phạm vi Trong phiên làm việc hiện tại Xuyên suốt các lần mở app
Dung lượng Giới hạn bởi RAM Lên đến 6MB (mặc định)
Kiểu dữ liệu Bất kỳ (object, array, function...) Chỉ string (cần JSON.stringify/parse)
Tóm gọn: Context API = "bảng trắng trong phòng họp" — mọi người đều thấy nhưng xóa khi ra về. AsyncStorage = "sổ tay cá nhân" — ghi vào thì giữ mãi dù đóng app.

2. Khi nào nên dùng Context API?

Context API phù hợp khi cần chia sẻ state giữa nhiều component mà không muốn truyền props qua nhiều tầng (prop drilling):

  • Thông tin người dùng đang đăng nhập (trong phiên hiện tại)
  • Theme (dark/light mode) — thay đổi ngay lập tức trên toàn app
  • Giỏ hàng tạm thời trong phiên mua sắm
  • Ngôn ngữ hiển thị (i18n)
  • Trạng thái loading/error toàn cục
Ví dụ trong bài: Danh sách yêu thích và thông tin user được lưu trong AppContext. Khi thêm sản phẩm ở DetailScreen, FavoritesScreen cập nhật ngay lập tức vì cùng dùng chung context. 
// Tạo context
const AppContext = createContext();

// Cung cấp dữ liệu
<AppProvider>
  <App />
</AppProvider>

// Sử dụng ở bất kỳ component con nào
const { favorites, addFavorite } = useAppContext();

3. Khi nào cần AsyncStorage?

AsyncStorage cần thiết khi dữ liệu phải tồn tại sau khi tắt và mở lại app:

  • Token đăng nhập (JWT) — để không phải đăng nhập lại mỗi lần mở app
  • Cài đặt người dùng (theme, ngôn ngữ, thông báo)
  • Lịch sử tìm kiếm, lịch sử xem
  • Dữ liệu offline (cache sản phẩm, bài viết)
  • Trạng thái onboarding (đã xem hướng dẫn chưa)
Ví dụ trong bài: Danh sách yêu thích lưu bằng AsyncStorage. Khi tắt app và mở lại, danh sách vẫn còn nguyên. Dùng useFocusEffect để tải lại dữ liệu mỗi khi tab được focus. 
// Lưu dữ liệu
await AsyncStorage.setItem('@favorites', JSON.stringify(list));

// Đọc dữ liệu
const json = await AsyncStorage.getItem('@favorites');
const list = json ? JSON.parse(json) : [];

// Xóa dữ liệu
await AsyncStorage.removeItem('@favorites');

4. Có thể kết hợp 2 cách không? Nếu có, giải thích cách làm.

Hoàn toàn có thể và đây là cách làm phổ biến nhất trong thực tế.

Chiến lược kết hợp: dùng AsyncStorage để khởi tạo Context, và Context để phân phối dữ liệu trong app.

Cách hoạt động:

  1. Khi app khởi động, đọc dữ liệu từ AsyncStorage vào state của Context
  2. Trong suốt phiên làm việc, các component đọc/ghi qua Context (nhanh, đồng bộ)
  3. Mỗi khi state thay đổi, tự động ghi lại vào AsyncStorage (để lưu vĩnh viễn)
export function AppProvider({ children }) {
  const [favorites, setFavorites] = useState([]);

  // Bước 1: Đọc từ AsyncStorage khi khởi động
  useEffect(() => {
    AsyncStorage.getItem('@favorites').then((json) => {
      if (json) setFavorites(JSON.parse(json));
    });
  }, []);

  // Bước 2: Mỗi khi favorites thay đổi, lưu vào AsyncStorage
  useEffect(() => {
    AsyncStorage.setItem('@favorites', JSON.stringify(favorites));
  }, [favorites]);

  // Bước 3: Cung cấp qua Context như bình thường
  const addFavorite = (item) => setFavorites(prev => [...prev, item]);

  return (
    <AppContext.Provider value={{ favorites, addFavorite }}>
      {children}
    </AppContext.Provider>
  );
}

Lợi ích của cách kết hợp:

Lợi ích Giải thích
Dữ liệu bền vững Nhờ AsyncStorage, dữ liệu không mất khi tắt app
Truy cập nhanh Nhờ Context, component đọc dữ liệu đồng bộ, không cần async/await
Code gọn hơn Component không cần biết về AsyncStorage, chỉ cần dùng useContext
Dễ bảo trì Logic lưu trữ tập trung trong Provider, không rải rác khắp nơi
Kết luận: Context API và AsyncStorage không phải là hai lựa chọn đối lập mà là hai công cụ bổ sung cho nhau. Trong ứng dụng thực tế, nên dùng AsyncStorage để khởi tạo và đồng bộ dữ liệu, còn Context API để phân phối dữ liệu đó đến các component một cách hiệu quả.
